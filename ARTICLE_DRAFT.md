## INFATON MCP Server — 35 инструментов для AI-интеграции с 1С:Предприятие

### Введение

Model Context Protocol (MCP) стремительно становится стандартом взаимодействия AI-ассистентов с внешними системами. Claude, GPT, Cursor, Windsurf — все они уже поддерживают MCP. Но у 1С-сообщества до сих пор не было готового решения.

Мы в **INFATON** выпустили open-source MCP-сервер для 1С:Предприятие — **35 инструментов**, которые позволяют любому AI-ассистенту работать с базой 1С как с родной.

**Репозиторий**: [github.com/infaton/MCP35](https://github.com/infaton/MCP35)

---

### Что такое MCP и зачем он 1С-нику

MCP (Model Context Protocol) — открытый протокол от Anthropic, описывающий как AI-модель может вызывать внешние инструменты. Протокол построен на JSON-RPC 2.0 и определяет три ключевых метода:

- `initialize` — рукопожатие между клиентом и сервером
- `tools/list` — получение списка доступных инструментов
- `tools/call` — вызов конкретного инструмента с параметрами

**Зачем это 1С-нику?** Представьте: вы открываете Claude Desktop (или Cursor, или любой MCP-совместимый клиент), подключаете к нему вашу базу 1С — и можете на естественном языке:

- «Покажи все документы РеализацияТоваровУслуг за март»
- «Какие реквизиты у справочника Номенклатура?»
- «Создай нового контрагента ООО Рога и Копыта»
- «Выполни запрос: ВЫБРАТЬ Номенклатура.Наименование ИЗ Справочник.Номенклатура»

AI-ассистент сам определит, какой инструмент вызвать, сформирует параметры и вернёт результат.

---

### Архитектура решения

INFATON MCP Server реализован как **расширение конфигурации 1С** (`.cfe`). Это значит:

- Не требует модификации основной конфигурации
- Устанавливается за 2 минуты через Конфигуратор
- Работает на любой конфигурации: ERP 2.5, УПП 1.3, Бухгалтерия 3.0, УТ 11, КА 2
- Поддерживает платформу 1С 8.3.20 и выше

```
┌─────────────────────────┐
│   AI Assistant          │  Claude / GPT / Cursor / Windsurf
│   (MCP Client)          │
└────────┬────────────────┘
         │ JSON-RPC 2.0 (HTTP)
         ▼
┌─────────────────────────┐
│   HTTP-сервис /hs/mcp/  │  Расширение INFATON_MCP.cfe
│   Module.bsl            │  35 инструментов
│   (MCP Server)          │
└────────┬────────────────┘
         │ Встроенный язык 1С
         ▼
┌─────────────────────────┐
│   База 1С:Предприятие   │  ERP / УПП / Бухгалтерия / ...
│   Данные, метаданные    │
└─────────────────────────┘
```

HTTP-сервис `mcp` публикуется по стандартному пути `/hs/mcp/` и принимает POST-запросы с Basic Auth. Никаких дополнительных серверов, прокси или middleware — всё работает нативно внутри 1С.

### 35 инструментов: полный каталог

#### Метаданные (8 инструментов)

Полный доступ к конфигурации «изнутри» — то, что раньше можно было делать только в Конфигураторе:

| # | Инструмент | Что делает |
|---|-----------|----------|
| 1 | `get_metadata_tree` | Дерево всех объектов конфигурации: справочники, документы, регистры, перечисления |
| 2 | `get_object_metadata` | Полная структура объекта: реквизиты, табличные части, типы данных |
| 3 | `get_object_attributes` | Список реквизитов с типами |
| 4 | `get_object_tabular_sections` | Табличные части и их колонки |
| 5 | `get_enum_values` | Значения перечисления |
| 6 | `get_register_dimensions` | Измерения, ресурсы и реквизиты регистра |
| 7 | `get_document_movements` | По каким регистрам двигается документ |
| 8 | `search_metadata` | Поиск объектов по имени/синониму |

AI-ассистент может самостоятельно изучить структуру вашей конфигурации, не заглядывая в Конфигуратор.

#### Данные (7 инструментов)

Чтение данных — от произвольных запросов до поиска по коду/наименованию:

| # | Инструмент | Что делает |
|---|-----------|----------|
| 9 | `execute_query` | Выполнение произвольного запроса на языке 1С |
| 10 | `get_object_by_ref` | Получение объекта по ссылке/GUID |
| 11 | `get_list` | Список элементов с фильтрами и пагинацией |

#### CRUD (7 инструментов)

### Быстрый старт

#### 1. Установка расширения

```
Конфигуратор → Конфигурация → Расширения конфигурации → Добавить → INFATON_MCP.cfe → F7
```

#### 2. Проверка

```bash
curl -u 'Логин:Пароль' -X POST \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"initialize","id":1}' \
  http://ваш-сервер/база/hs/mcp/
```

Ответ:
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "protocolVersion": "2024-11-05",
    "serverInfo": { "name": "infaton-1c-mcp", "version": "1.0.0" }
  }
}
```

#### 3. Пример вызова

```bash
curl -u 'Логин:Пароль' -X POST \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"tools/call","id":2,"params":{"name":"get_metadata_tree","arguments":{}}}' \
  http://ваш-сервер/база/hs/mcp/
```

### Практические сценарии

**Аналитик 1С в Claude Desktop:**
> «Покажи структуру документа РеализацияТоваровУслуг — все реквизиты и табличные части»

Claude вызывает `get_object_metadata` → `get_object_tabular_sections` и возвращает полную структуру с описаниями полей.

**Разработчик в Cursor/Windsurf:**
> «Напиши обработку, которая находит все непроведённые документы ПоступлениеТоваровУслуг за 2025 год»

Ассистент сначала изучает структуру через `get_object_metadata`, потом генерирует код, проверяет через `execute_query` — и выдаёт готовую обработку.

**DevOps:**
> «Есть ли блокировки? Сколько пользователей сейчас в базе?»

`get_locks` + `get_active_users` — моментальный ответ без входа в 1С.

### Безопасность

- **Basic Auth** — стандартная аутентификация 1С, пароли не хранятся в MCP-сервере
- **Права доступа 1С** — инструменты работают в контексте пользователя, чьи логин/пароль указаны. Нет прав на справочник — инструмент вернёт ошибку
- **execute_code** — самый мощный инструмент, журналируется в ЖурналРегистрации (событие `INFATON.ExecuteCode`)
- **Расширение** — не модифицирует основную конфигурацию, легко отключается

## INFATON — AI-платформа для внедрения 1С:ERP

INFATON MCP Server — это лишь один из компонентов экосистемы **INFATON** — платформы, которая использует искусственный интеллект для автоматизации полного цикла внедрения 1С:ERP.

### Полный цикл автоматического обследования

Традиционное обследование перед внедрением ERP — это недели интервью, горы протоколов, субъективные оценки аналитиков. INFATON автоматизирует этот процесс с помощью трёх независимых AI-движков:

- **TwinAgent (v1)** — глубокий агентный анализ через MCP. LLM самостоятельно решает, какие метаданные и данные исследовать. Максимальное качество отчёта.
- **Pipeline (v2)** — быстрый MapReduce. Полный каталог объектов + пакетный анализ + синтез. Максимальная скорость.
- **Expert Swarm (v3)** — рой микро-экспертов. Каждый объект 1С анализируется отдельным AI-агентом, результаты объединяются. Баланс между качеством и скоростью.

Все три движка дополняют друг друга: v1 ищет глубинные инсайты, v2 обеспечивает полноту покрытия, v3 даёт детальный пообъектный анализ. Итоговый **гибридный отчёт** объединяет результаты всех трёх.

### Цифровые двойники

Цифровой двойник в INFATON — это не дашборд и не копия базы. Это **AI-агент с конкретной ролью**, привязанный к подсистемам 1С.

Пример: двойник «Главный бухгалтер» знает структуру подсистемы Бухгалтерия, видит только свои OData-сущности, и может:
- В режиме **Советник** — анализировать данные, отвечать на вопросы, формировать рекомендации
- В режиме **Автопилот** — самостоятельно выполнять действия: проводить документы, создавать записи, запускать отчёты

#### Три режима работы двойников

1. **💡 Советник (Advisor)** — human-in-the-loop. Двойник анализирует, рекомендует, но не вносит изменения. Все решения остаются за человеком. Идеален для этапа обследования и первых месяцев эксплуатации.

2. **🤖 Автопилот (Autopilot)** — автономный режим. Двойник выполняет действия самостоятельно по заданным правилам и политикам безопасности. Все действия логируются и могут быть отменены.

3. **⚡ Оба режима (Both)** — двойник поддерживает переключение между Советником и Автопилотом. Оператор в любой момент может перевести двойника из одного режима в другой.

Фабрика агентов позволяет создать команду из 25-50 двойников за минуты — по одному на каждую ключевую роль в проекте внедрения.

### Закрытый контур и локальные LLM

**Это ключевое преимущество INFATON.** Платформа спроектирована для работы в трёх режимах окружения:

#### ☁️ Облачный (Cloud)
Облачные LLM (GPT, Claude, Gemini), MCP через интернет. Для быстрого старта и пилотных проектов.

#### 🔀 Гибридный (Hybrid)
Локальные LLM (Ollama, LM Studio, vLLM) для основной нагрузки + облачные для сложных задач. Чувствительные данные не покидают периметр.

#### 🔒 Закрытый контур (Airgapped)
**Полная автономность без интернета:**
- Только локальные LLM (Qwen, LLaMA, Mistral и др.)
- MCP-подключения только по LAN/VPN — публичные адреса заблокированы
- Никаких cloud-запросов
- Платформа автоматически определяет режим и переключает функционал

Для предприятий с жёсткими требованиями ИБ — банки, оборонка, госсектор, критическая инфраструктура — это единственный приемлемый вариант. INFATON позволяет получить все преимущества AI-автоматизации **без компромиссов по безопасности данных**.

Админ может зафиксировать режим принудительно или доверить автодетекцию — платформа сама определяет доступность интернета, облачных ключей и локальных LLM-серверов.

### Ещё возможности платформы

- **Трассировка проблем** — AI-анализ интервью и документов, автоматическое извлечение проблем с 5W+5Why, приоритизация по WSJF
- **GAP-анализ** — матрица расхождений «как есть» vs «как надо» с привязкой к подсистемам ERP
- **База знаний** — гибридный RAG-поиск (FTS + триграммы + LLM-ранжирование) по документации, стандартам, шаблонам
- **MCP-интеграция** — те самые 35 инструментов для прямого доступа AI к данным 1С
- **Зеркало** — AI-симуляция диалогов между цифровыми двойниками для выявления противоречий в бизнес-процессах

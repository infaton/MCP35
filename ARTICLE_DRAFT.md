## INFATON MCP Server v2.1 — 51 инструмент для AI-интеграции с 1С:Предприятие

### Введение

Model Context Protocol (MCP) стремительно становится стандартом взаимодействия AI-ассистентов с внешними системами. Claude, GPT, Cursor, Windsurf — все они уже поддерживают MCP. Но у 1С-сообщества до сих пор не было готового решения.

Мы в **INFATON** выпустили open-source MCP-сервер для 1С:Предприятие. За месяц проект вырос с 35 до **51 инструмента** — добавились бухгалтерия, CDC (отслеживание изменений), пакетные операции, аудит и многое другое.

**Репозиторий**: [github.com/infaton/MCP35](https://github.com/infaton/MCP35)  
**Каталог Glama.ai**: [glama.ai/mcp/servers/infaton/MCP35](https://glama.ai/mcp/servers/infaton/MCP35)  
**Каталог awesome-mcp-servers**: включён ✅

### Что такое MCP и зачем он 1С-нику

MCP (Model Context Protocol) — открытый протокол от Anthropic, описывающий как AI-модель может вызывать внешние инструменты. Протокол построен на JSON-RPC 2.0 и определяет три ключевых метода:

- `initialize` — рукопожатие между клиентом и сервером
- `tools/list` — получение списка доступных инструментов
- `tools/call` — вызов конкретного инструмента с параметрами

**Зачем это 1С-нику?** Представьте: вы открываете Claude Desktop (или Cursor, или любой MCP-совместимый клиент), подключаете к нему вашу базу 1С — и можете на естественном языке:

- «Покажи остатки по счёту 41.01 за первый квартал» → `get_balance`
- «Какие реквизиты у справочника Номенклатура?» → `get_object_metadata`
- «Создай нового контрагента и заполни на основании договора» → `create_object` + `fill_on_basis`
- «Покажи все изменения в справочниках за последний час» → `get_changes_since`
- «Проверь документ перед проведением» → `validate_document`
- «Найди дубли в справочнике Контрагенты» → `find_duplicates`

AI-ассистент сам определит, какой инструмент вызвать, сформирует параметры и вернёт результат.

#### Ключевые багфиксы в v2.1

- **`get_list`** — добавлен `ПЕРВЫЕ N` (раньше мог вернуть ВСЕ записи!)
- **`get_object_by_ref`** — корректная обработка пустых ссылок
- **`get_active_users`/`get_locks`** — исправлен обход коллекции сеансов
- **`get_event_log`** — полная перезапись парсинга дат (ISO 8601 + 3 формата)
- **`fill_on_basis`** — исправлен вызов метода `ЗаполнитьНаОсновании()`
- **`execute_batch`** — защита от рекурсии (guard-флаг `_batch_depth`)

### Архитектура решения

INFATON MCP Server реализован как **расширение конфигурации 1С** (`.cfe`). Это значит:

- Не требует модификации основной конфигурации
- Устанавливается за 2 минуты через Конфигуратор
- Работает на любой конфигурации: ERP 2.5, УПП 1.3, Бухгалтерия 3.0, УТ 11, КА 2
- Поддерживает платформу 1С 8.3.20 и выше

```
┌─────────────────────────┐
│   AI Assistant          │  Claude / GPT / Cursor / Windsurf
│   (MCP Client)          │
└────────┬────────────────┘
         │ stdio (JSON-RPC 2.0)
         ▼
┌─────────────────────────┐
│   index.mjs             │  Node.js stdio → HTTP proxy
│   (MCP Transport)       │  (входит в репозиторий)
└────────┬────────────────┘
         │ HTTP POST (Basic Auth)
         ▼
┌─────────────────────────┐
│   HTTP-сервис /hs/mcp/  │  Расширение INFATON_MCP.cfe
│   Module.bsl            │  51 инструмент (4 263 строки)
│   (MCP Server)          │
└────────┬────────────────┘
         │ Встроенный язык 1С
         ▼
┌─────────────────────────┐
│   База 1С:Предприятие   │  ERP / УПП / Бухгалтерия / УТ / КА
│   Данные, метаданные    │
└─────────────────────────┘
```

В v2.0 появился **index.mjs** — Node.js stdio-обёртка, позволяющая подключать MCP-сервер через стандартный stdio-транспорт (используется в Claude Desktop, Cursor и других клиентах). HTTP-сервис `mcp` принимает POST-запросы с Basic Auth — никаких дополнительных серверов не требуется.

### Быстрый старт

#### 1. Установка расширения

```
Конфигуратор → Конфигурация → Расширения конфигурации → Добавить → INFATON_MCP.cfe → F7
```

#### 2. Проверка подключения

```bash
curl -u 'Логин:Пароль' -X POST \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"initialize","id":1}' \
  https://ваш-сервер/база/hs/mcp/
```

Ответ:
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "protocolVersion": "2024-11-05",
    "capabilities": { "tools": {} },
    "serverInfo": { "name": "infaton-1c-mcp", "version": "2.1.0" }
  }
}
```

#### 3. Подключение к Claude Desktop / Cursor

В конфиге MCP-клиента (`claude_desktop_config.json` или аналогичном):

```json
{
  "mcpServers": {
    "infaton-1c": {
      "command": "node",
      "args": ["path/to/MCP35/index.mjs"],
      "env": {
        "ONEC_BASE_URL": "https://ваш-сервер/база/hs/mcp/",
        "ONEC_USERNAME": "Логин",
        "ONEC_PASSWORD": "Пароль"
      }
    }
  }
}
```

#### 4. Пример: остатки по счёту

```bash
curl -u 'Логин:Пароль' -X POST \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"tools/call","id":2,"params":{"name":"get_balance","arguments":{"account_code":"41.01","period_from":"2026-01-01T00:00:00","period_to":"2026-03-31T23:59:59"}}}' \
  https://ваш-сервер/база/hs/mcp/
```

### Практические сценарии

**Главный бухгалтер в Claude Desktop:**
> «Покажи остатки по счёту 41.01 за первый квартал 2026 и проводки по последнему документу реализации»

Claude вызывает `get_balance` → `get_document_list` → `get_accounting_entries` и формирует сводку с дебетовыми/кредитовыми оборотами.

**Аналитик 1С:**
> «Покажи структуру документа РеализацияТоваровУслуг — все реквизиты и табличные части»

Claude вызывает `get_object_metadata` → `get_object_tabular_sections` и возвращает полную структуру.

**Разработчик в Cursor:**
> «Найди все изменения в справочнике Контрагенты за последний час и проверь, нет ли дублей»

Ассистент вызывает `get_changes_since` → `find_duplicates` и показывает результат с рекомендациями.

**DevOps / Администратор:**
> «Есть ли блокировки? Какие расширения установлены? Какие права у пользователя Иванов?»

`get_locks` + `get_configuration_extensions` + `get_rights` — моментальный ответ без входа в 1С.

**Миграция данных:**
> «Создай документ ПоступлениеТоваровУслуг и заполни на основании заказа поставщику №123»

`create_object` + `fill_on_basis` — AI-ассистент обрабатывает миграцию документов между базами.

### Безопасность

- **Basic Auth** — стандартная аутентификация 1С
- **Права доступа 1С** — инструменты работают в контексте пользователя. Нет прав — инструмент вернёт ошибку
- **execute_code** — журналируется в ЖурналРегистрации (событие `INFATON.ExecuteCode`)
- **execute_batch** — защита от рекурсии (guard-флаг `_batch_depth`)
- **Расширение** — не модифицирует конфигурацию, легко отключается

## INFATON — AI-платформа для внедрения 1С:ERP

INFATON MCP Server — это лишь один из компонентов экосистемы **INFATON** — платформы, которая использует искусственный интеллект для автоматизации полного цикла внедрения 1С:ERP.

### Полный цикл автоматического обследования

Традиционное обследование перед внедрением ERP — это недели интервью, горы протоколов, субъективные оценки аналитиков. INFATON автоматизирует этот процесс с помощью трёх независимых AI-движков:

- **TwinAgent (v1)** — глубокий агентный анализ через MCP. LLM самостоятельно решает, какие метаданные и данные исследовать. Максимальное качество отчёта.
- **Pipeline (v2)** — быстрый MapReduce. Полный каталог объектов + пакетный анализ + синтез. Максимальная скорость.
- **Expert Swarm (v3)** — рой микро-экспертов. Каждый объект 1С анализируется отдельным AI-агентом, результаты объединяются.

Итоговый **гибридный отчёт** объединяет результаты всех трёх движков.

### Цифровые двойники

Цифровой двойник в INFATON — это **AI-агент с конкретной ролью**, привязанный к подсистемам 1С.

Пример: двойник «Главный бухгалтер» знает структуру подсистемы Бухгалтерия, видит только свои данные, и может:
- В режиме **Советник** — анализировать данные, формировать рекомендации
- В режиме **Автопилот** — самостоятельно выполнять действия через MCP

Фабрика агентов позволяет создать команду из 25-50 двойников за минуты.

### Закрытый контур и локальные LLM

**Ключевое преимущество INFATON.** Три режима работы:

- ☁️ **Облачный** — GPT, Claude, Gemini. Для быстрого старта.
- 🔀 **Гибридный** — локальные LLM + облачные для сложных задач.
- 🔒 **Закрытый контур** — только локальные LLM, MCP по LAN/VPN, никаких cloud-запросов.

Для предприятий с жёсткими требованиями ИБ — банки, оборонка, госсектор — это единственный приемлемый вариант.
